# UIUbuntuXkeen

- Текущая версия линии: **v0.6.52**
- Базовый runtime текущего хоста: `mihomo`
- Подтверждённый recovery-механизм UI: остановить `mihomo`, удалить папку UI, снова запустить `mihomo` — UI скачивается из репозитория автоматически
- Начиная с `v0.6.52`, в репозитории снова есть **отдельный Ubuntu backend**, но он запускается **как самостоятельный systemd service**, а не внутри `mihomo`

## Что в v0.6.52

- backend `ultra-ui-ubuntu-backend.service` остаётся подтверждённым на живом Ubuntu-хосте и доступен по `0.0.0.0:18090`
- экран `Хосты 3x-ui` теперь даёт два явных server-side действия: `Проверить сейчас` и `Обновить SSL-кэш`
- карточка SSL-провайдеров тоже умеет запускать проверку и refresh кэша прямо из UI
- install/update backend теперь не требует отдельного ручного `systemctl restart`: это делает `backend/scripts/install.sh`
- документация, transfer-файлы и базы шагов/запросов обновлены под подтверждённый live-backend и SSL-action polish

## Как это теперь запускается

- UI по-прежнему живёт отдельно и обслуживается `mihomo`
- backend ставится и запускается отдельно, как `ultra-ui-ubuntu-backend.service`
- UI подключается к нему через `Setup`, например на `http://192.168.5.23:18090`
- активный backend-режим на Ubuntu-хосте: **`ubuntu-service`**

## Важно

Этот релиз **не вшивает backend внутрь `mihomo`** и не пытается воскресить мёртвый `/cgi-bin/api.sh`.
Новая backend-линия запускается отдельно и может развиваться независимо, не ломая текущий UI runtime.
