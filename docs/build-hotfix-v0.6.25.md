# Build hotfix audit — v0.6.25

## Что исправлено

- `Preflight diagnostics` перенесён перед чтением версии проекта, чтобы при раннем падении workflow сразу было видно `pwd`, `ls -la`, версии `node` и `pnpm`.
- `Read project version` больше не использует хрупкий inline `node -p` шаблон; теперь версия читается через `node:fs` и `JSON.parse`.
- `Show build log on failure` больше не падает вторично, если `build-ui.log` ещё не создан.

## Зачем это нужно

Предыдущий workflow мог умереть до стадии `vite build`, а в интерфейсе GitHub Actions это выглядело как пустой `Error:` без полезного лога. После этого hotfix следующий прогон либо пройдёт, либо покажет уже конкретную раннюю причину падения.
