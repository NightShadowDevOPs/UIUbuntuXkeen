# Следующий шаг после v0.6.43

## Ближайшая цель
Сделать **v0.6.44** как точечный audit/research релиз server-side контура проекта без ломки UI.

## Что именно проверить
1. Какие части текущего проекта реально живут в `api.sh` и могут считаться существующим backend contour.
2. Какие capability-флаги действительно доступны на живом Ubuntu/Mihomo runtime.
3. Где в существующем контуре можно безопасно хранить:
   - список `3x-ui Hosts`;
   - даты окончания сертификатов и время последней проверки;
   - таблицу `Users` (`IP`, `MAC`, `hostname`, `proxyAccess`).
4. Какие UI-экраны уже готовы к подключению этого backend без повторного изобретения отдельного service.

## Чего не делать
- не добавлять новый standalone service;
- не подменять фактический runtime проекта предположениями;
- не менять подтверждённый recovery/update path через `mihomo`.
