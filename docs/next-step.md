# Следующий шаг после v0.6.44

## Ближайшая цель
Сделать **v0.6.45** как серверно-прикладной релиз для фактического применения `proxyAccess` и полировки хранения SSL-метаданных в существующем contour проекта.

## Что именно делать
1. Проверить, какие текущие команды `api.sh` уже применяют ограничения доступа к proxy-портам и как безопасно связать их с таблицей `Пользователи`.
2. Довести серверное применение `proxyAccessPolicyMode` и индивидуального `proxyAccess` без влияния на остальной сетевой доступ клиента.
3. Сверить, какие SSL-поля уже реально возвращает `mihomo_providers`, и при необходимости расширить выдачу без выдумывания нового backend service.
4. Добавить smoke-check для `Хосты 3x-ui` и `Пользователи` на живом контуре Mihomo/UI.

## Чего не делать
- не добавлять новый standalone service;
- не ломать подтверждённый recovery/update path через `mihomo`;
- не смешивать Ubuntu-проект с роутерной веткой.
