# База запросов на изменения

Актуально для релиза: **v0.6.47**

## Запросы пользователя
- Если backend на текущем хосте ещё не работает, UI не должен притворяться, что всё уже подключено.
- Нужен честный режим без фальшивых capability fallback и без `Network Error` как будто backend почти живой.
- Подготовить полный архив для переноса проекта в новый чат.

## Реализация в v0.6.47
- UI больше не включает compatibility-bridge fallback автоматически, если `/api/capabilities` и `/cgi-bin/api.sh` реально отсутствуют.
- `Хосты 3x-ui` показывают предупреждение о недоступном backend route и продолжают работать с локальными URL панелей.
- `Пользователи` показывают такое же предупреждение и работают с локальными данными UI.
- Подготовлен обновлённый transfer-архив с фиксацией реального состояния проекта.
