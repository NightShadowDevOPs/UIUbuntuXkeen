# База запросов на изменения

Актуально для релиза: **v0.6.46**

## Запросы, зафиксированные на этом шаге
- продолжить проект от стабилизированной базы `v0.6.44`;
- сделать так, чтобы backend перестал быть фантомом и реально заработал в существующем contour проекта;
- не создавать новый отдельный service;
- продолжать фиксировать каждый запрос и каждый шаг в документах проекта.

## Решение в v0.6.45
- включён рабочий capability fallback через существующий `api.sh`;
- добавлена команда `cmd=capabilities` в server-side bridge `api.sh`;
- экраны `Хосты 3x-ui` и `Пользователи` переключены на реальный contour `Mihomo + api.sh + shared users DB`;
- отдельный новый service не создавался.

## Открытые вопросы
- server-side применение `proxyAccess` как отдельная backend-команда;
- при необходимости — отдельные точечные команды `api.sh` для `3x-ui Hosts` и `Users`, но без нового сервиса.

## Дополнение в v0.6.46
- Исправить пустой экран/пустой список в `Хосты 3x-ui`, если адреса панелей уже есть в `Прокси-провайдерах`.
