# База запросов на изменения

Актуально для релиза: **v0.6.51**

## Запросы пользователя
- Продолжить проект здесь и начать делать backend.
- Объяснить, как backend будет запускаться.
- Делать релизы поэтапно и по ходу объяснять, что делать пользователю на его стороне.
- Постоянно актуализировать документацию.
- Фиксировать все шаги и все запросы на изменения.
- Не предлагать workflow с копированием архива на сервер для каждого обновления; перейти на стабильную папку `/opt/UIUbuntu` и `git pull` от клонированного репозитория.
- Перед выдачей релиза многократно прогонять install/update сценарии.
- После живого запуска backend исправить экранные баги интеграции, а не отправлять пользователя обратно в другой режим.

## Реализация в v0.6.49
- backend стартует как отдельный Ubuntu service, а не как часть `mihomo` и не как `cgi-bin/api.sh`
- в репозиторий добавлен отдельный каталог `backend/` с готовым FastAPI + SQLite MVP
- добавлены install/dev-run сценарии и systemd unit template
- документация и transfer-файлы обновлены под новый backend-этап
- шаги работ и пользовательские запросы зафиксированы в `docs/worklog-db.md` и `docs/change-requests-db.md`

## Дополнение v0.6.49

- Требование пользователя: перед выдачей релиза многократно прогонять install/update сценарии, а не только собирать архив.
- В ответ на это `v0.6.49` исправил сценарий установки и подготовил стабильный server workflow через `/opt/UIUbuntu/app`.
