# База запросов на изменения

Актуально для релиза: **v0.6.43**

## Запросы, зафиксированные на этом шаге
- продолжить проект от очищенной базы без новых фантомных сущностей;
- зафиксировать реальный runtime contour проекта и перестать тащить в логику выдуманный service;
- сохранить отдельные разделы `Хосты 3x-ui` и `Пользователи`, но не ломать старт UI;
- убрать из дистрибутива и интерфейса хвосты, создающие ложное впечатление о чужой архитектуре;
- продолжать сохранять каждый запрос и каждый шаг в документах проекта.

## Решение в v0.6.43
- корневой маршрут UI переведён на `Overview`;
- оформлен аудит backend-контура и recovery/update flow через `mihomo`;
- видимые тексты на новых разделах очищены от упоминаний отдельного фантомного service;
- следующий шаг ограничен audit/research внутри существующего backend-контура проекта.

## Открытые вопросы
- где именно в существующем server-side contour хранить данные `3x-ui Hosts` и `Users`;
- как подключить серверную SSL-проверку без появления новой искусственной сущности.
