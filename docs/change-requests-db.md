# База запросов на изменения

Актуально для релиза: **v0.6.44**

## Запросы, зафиксированные на этом шаге
- продолжить проект от стабилизированной базы `v0.6.43`;
- разобрать реальный `api.sh`/capabilities-контур проекта;
- подключить `Хосты 3x-ui` и `Пользователи` к существующему backend-слою проекта без нового service;
- продолжать фиксировать каждый запрос и каждый шаг в документах проекта.

## Решение в v0.6.44
- compatibility bridge теперь объявляет реально существующие возможности `api.sh` для SSL-проверок и shared users DB;
- `Хосты 3x-ui` читают и сохраняют `providerPanelUrls` через `users_db_get` / `users_db_put`;
- `Пользователи` читают и сохраняют LAN inventory и `proxyAccessPolicyMode` через тот же shared users DB;
- `mihomo_providers` и `ssl_cache_refresh` используются как фактический серверный контур для SSL-состояния провайдеров.

## Открытые вопросы
- как именно применить `proxyAccess` к proxy-портам на сервере без влияния на остальной трафик;
- нужно ли расширять выдачу `mihomo_providers` дополнительными SSL-полями или текущих уже достаточно.
