# v0.6.40 — cleanup phantom service

Ключевой смысл релиза: убрать из дистрибутива искусственно добавленный отдельный `ubuntu-service/`, который не существовал на сервере пользователя и создал ложную архитектурную ветку.

Что сделано:
- удалён каталог `ubuntu-service/`;
- убраны текущие инструкции по systemd/service deployment;
- UI отвязан от жёсткого ожидания `ubuntu-service` как обязательного режима;
- документация обновлена так, чтобы следующий backend-шаг шёл только через существующий backend проекта.
