# v0.6.70 — backend UI routes + SSL status clarity

## Что исправлено
- backend `ubuntu-service` теперь проксирует дополнительные Mihomo endpoints, которые использует UI (`/group/*`, `/cache/*`, `/restart`, `/upgrade/ui`, `/configs/*`), чтобы backend-контур не упирался в локальные `Network Error` на служебных действиях;
- bridge к Mihomo теперь безопаснее прокладывает пустые `PUT/POST/PATCH` и не валится traceback'ом на неожиданных исключениях;
- SSL-статусы в `Хосты 3x-ui` и в workspace проверки сертификатов получили более контрастные тёмные плашки и короткие подписи вместо каши из длинных ошибок;
- UI теперь явно подписывает, что серверная TLS-проверка провайдеров идёт `system route` текущего Ubuntu-хоста, а не через выбранную proxy-group пользователей.

## Практический эффект
- backend UI стабильнее работает в контуре `ubuntu-service` и перестаёт выглядеть «полуживым», когда фронт просит стандартные Mihomo actions;
- TLS timeout теперь читается как проблема маршрута/рукопожатия именно с этого хоста, а не как «сертификат внезапно плохой везде».
