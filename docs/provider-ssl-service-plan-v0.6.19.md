# Provider SSL service plan — v0.6.19

## Что фиксируем

Проверка сертификатов Провайдеров в UIUbuntuXkeen должна выполняться **на Ubuntu-хосте этого проекта**, а не в браузере и не через legacy router-agent контур.

## Что остаётся в UI

- список Провайдеров;
- редактирование URL 3x-ui подписок;
- локальная визуализация сохранённых URL и порогов предупреждения;
- запуск серверной проверки и чтение кэша после появления Ubuntu service.

## Что должен делать server-side слой

- хранить общий список URL подписок Провайдеров на хосте;
- выполнять TLS/SSL-проверку по каждому URL отдельно;
- не валить весь цикл, если один Провайдер недоступен;
- сохранять notAfter / issuer / subject / SAN / ошибку / checkedAt;
- иметь scheduler/timer и ручной refresh;
- отдавать результаты UI через `/api/providers/checks` и связанные endpoints.

## Рекомендуемый runtime

- отдельный `ultra-ui-ubuntu.service`;
- `ultra-ui-ubuntu.timer` для плановой проверки;
- runtime state в `/var/lib/ultra-ui-ubuntu/`;
- логи в `/var/log/ultra-ui-ubuntu/`.

## Граница ответственности

- `Прокси-провайдеры`: задают и редактируют URL 3x-ui подписок;
- `Задачи`: показывают operational-состояние проверки и запускают ручной refresh;
- backend/service: реально проверяет сертификаты и держит кэш/историю.

## Что меняет v0.6.19

- UI больше не притворяется, что provider SSL checks уже доступны через legacy agent/compatibility path;
- сохранение URL подписок остаётся доступным;
- в `Задачах` и связанных подсказках явно показано, что проверка появится после подключения Ubuntu service на этом хосте.
