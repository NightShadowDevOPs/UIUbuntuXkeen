# Текущий статус — v0.6.20

Дата: 2026-03-31

## Где мы сейчас

- Текущая линия: **v0.6.20**
- Последний подтверждённо рабочий релиз на сервере: **v0.2.10**
- Текущий подход: **cleanup-first + честная фиксация runtime-модели**

## Что зафиксировано этим релизом

- `v0.6.17` **не считается правильным релизом по архитектуре**;
- проект возвращён к модели `v0.6.16`: **статический frontend UI**, который подключается к выбранному backend через `Setup`;
- навязанная ветка `frontend + встроенный backend runtime` откатена;
- docs обновлены так, чтобы больше не создавать ложное впечатление, будто provider SSL checks уже закрыты внутри самого UI-проекта.

## Что это означает

Главный UI снова соответствует фактическому состоянию проекта.

Сейчас нельзя честно утверждать, что в проекте уже есть встроенный Ubuntu service, scheduler, SQLite runtime и server-side SSL engine для Провайдеров. Всё это остаётся **целевым следующим этапом**, а не закрытой реальностью.

## Что уже сделано в cleanup-волне

- `Home` больше не поднимает router legacy автоматически;
- `Хост` очищен до host/runtime экрана;
- `Трафик` очищен от compatibility/router карточек;
- `Tasks`, `Users`, `Policies` скрыты из основного меню и оставлены для legacy-аудита.

## Что остаётся проблемным

- `TasksPage.vue` — перегруженный legacy-control-room;
- `SubscriptionsPage.vue` — всё ещё требует пересборки под канонический server publish path;
- `UsersPage.vue` и `PoliciesPage.vue` — пока сохранены только для аудита старой логики;
- проверка SSL сертификатов Провайдеров — **ожидаемая продуктовая функция**, но в текущем runtime ещё не закрыта.


## v0.6.20

- закрыты текущие build/type-check blockers из CI/Actions (`uiBuild`, `UserTrafficStats`, `ProxyProvider`, `HostRuntimeCard`);
- `tsconfig.app.json` переведён на `ES2022`;
- продуктовая логика provider SSL не подменялась фальшивым frontend-only runtime.

## v0.6.19

- legacy provider SSL fallback через router-agent/compatibility path в UI больше не считается рабочим путём;
- редактирование URL 3x-ui подписок сохранено;
- место реальной проверки сертификатов зафиксировано: отдельный Ubuntu service на хосте этого проекта.
