# Текущий статус — v0.6.23

Дата: 2026-03-31

## Где мы сейчас

- Текущая линия: **v0.6.23**
- Последний подтверждённо рабочий релиз на сервере: **v0.2.10**
- Текущий подход: **cleanup-first + честная фиксация runtime-модели + нормальный CI-лог сборки**

## Что зафиксировано этим релизом

- В репозиторий добавлен `.github/workflows/build-ui.yml`.
- Шаг `Build UI` теперь печатает полный лог `vite build --debug`, а не только `Process completed with exit code 1`.
- При падении workflow содержимое `build-ui.log` выводится прямо в лог GitHub Actions.
- Upload artifacts намеренно не добавлялись.

## Что это означает

Главный эффект `v0.6.23` — дальше мы должны видеть **реальную причину падения сборки**, а не немой `exit code 1`.

Этот релиз не меняет продуктовую архитектуру и не закрывает новые фичи. Он нужен, чтобы следующий build-hotfix уже делался по фактической ошибке Vite/Rollup.

## Что остаётся важным по проекту

- `v0.6.17` **не считается правильным релизом по архитектуре**.
- Проект остаётся в модели **статический frontend UI + выбранный backend через Setup**.
- `TasksPage.vue` по-прежнему перегружен и требует cleanup по блокам.
- Проверка SSL сертификатов Провайдеров остаётся **целевой серверной функцией** на Ubuntu-хосте и ещё не считается закрытой во frontend-only runtime.
