# UIUbuntuXkeen cleanup wave 1 — v0.6.16

Дата: 2026-03-31

## Цель

Перестать смешивать UIUbuntuXkeen с роутерным проектом и убрать из основной навигации разделы, которые пока живут только за счёт legacy-логики.

## Что изменено

### Home
- убран автозапуск router legacy bootstrap при открытии UI;
- главная оболочка больше не делает вид, что старый роутерный контур обязателен для запуска проекта.

### Хост
- `RouterPage.vue` очищен до роли **«Хост»**;
- оставлены только секции:
  - Обзор
  - Сервисы
  - Логи
- удалены из основного host-экрана: `AgentCard`, `SystemCard`, `NetcrazeTrafficCard`, `NetworkCard`.

### Трафик
- удалены из основной страницы хостового трафика compatibility/router карточки QoS и Netcraze;
- страница оставлена как runtime-экран: клиенты, соединения, пользователи.

### Навигация
Из основного меню убраны разделы:
- `Задачи`
- `Пользователи`
- `Политики`

Они сохранены в коде только для поэлементного аудита legacy-логики.

## Почему именно так

Эти три раздела создавали ложное ощущение, что в Ubuntu/server-линии уже есть готовый backend execution path для:
- SSL checks
- users-db sync
- policy enforcement
- QoS/shaping
- topology/diagnostics jobs

На текущем этапе это не так. Поэтому в основном UI они больше не должны притворяться штатными operational-экранами.
