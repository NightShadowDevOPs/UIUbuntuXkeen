# Документация UIUbuntuXkeen

В этом проекте `docs/` — каноничный комплект документов, который должен обновляться **в каждом релизе**.

Актуально для релиза: **v0.6.22**

## Что здесь должно быть всегда

- `project-spec.md` — полное ТЗ и архитектурная модель проекта.
- `backend-contract.md` — технический каркас Ubuntu backend/service API, фоновых задач и хранения состояния.
- `functional-audit.md` — глубокий аудит функционального паритета: что было в Xkeen, что есть сейчас и что требует Ubuntu service.
- `roadmap.md` — план работ по релизам и статус этапов.
- `releases.md` — журнал выполненных работ по релизам.
- `chat-transfer.md` — актуальный пакет переноса проекта в новый чат.
- `current-status.md` — короткий и честный статус текущей линии.
- `next-step.md` — ближайший шаг без гаданий и самообмана.

## Дополнительные audit-документы

- `screen-audit-v0.6.13.md` — поэкранный срез интерфейса.
- `route-audit-v0.6.13.md` — срез по роутам и legacy-зависимостям.
- `cleanup-wave-1.md` — что уже вычищено из основной Ubuntu-навигации.
- `runtime-model-audit-v0.6.18.md` — фактическая runtime-модель проекта и почему `v0.6.17` был откатан.

## Правила ведения документации

1. Не плодить дублирующие transfer-файлы на каждый патч.
2. Не хранить в `docs/` устаревшие router-only инструкции, если они уже вошли в ТЗ, backend contract, audit или roadmap.
3. При каждом релизе обновлять минимум:
   - версию проекта;
   - текущий статус;
   - ближайшие шаги;
   - перенос в новый чат.
4. Если меняется release-flow, GitHub Actions или способ обновления через UI — отражать это в `README.md`, `docs/releases.md` и `docs/chat-transfer.md`.
5. Если меняется архитектура, пути, backend contract, план фоновых задач или модель хранения результатов — сначала обновлять `docs/`, потом код.
6. Если релиз оказался ошибочным архитектурно, это должно быть прямо зафиксировано в `current-status`, `releases`, `roadmap` и transfer-документах, а не замалчиваться.

- `provider-ssl-service-plan-v0.6.19.md` — честная схема, где и как должны проверяться сертификаты Провайдеров в Ubuntu-линии.
- `build-hotfix-v0.6.22.md` — разбор и фиксация build/type-check ошибок, которые уронили CI/Actions после `v0.6.19`.
