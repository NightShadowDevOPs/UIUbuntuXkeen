# Документация UIUbuntuXkeen

В этом проекте `docs/` — это каноничный комплект документов, который должен обновляться **в каждом релизе**.

## Что здесь должно быть всегда

- `project-spec.md` — полное ТЗ и архитектурная модель проекта.
- `roadmap.md` — план работ по релизам и статус этапов.
- `releases.md` — журнал выполненных работ по релизам.
- `chat-transfer.md` — актуальный пакет переноса проекта в новый чат.

## Правила ведения документации

1. Не плодить дублирующие transfer-файлы на каждый патч.
2. Не хранить в `docs/` устаревшие router-only инструкции, если они уже вошли в ТЗ или roadmap.
3. При каждом релизе обновлять минимум:
   - версию проекта;
   - текущий статус;
   - ближайшие шаги;
   - перенос в новый чат.
4. Если меняется release-flow, GitHub Actions или способ обновления через UI — это тоже считается частью каноничной документации и должно быть отражено в `README.md`, `docs/releases.md` и `docs/chat-transfer.md`.
4. Если меняется архитектура, пути, рабочий процесс или release flow — сначала обновлять `docs/`, потом код.

## Базовый состав релизной документации

Каждый релиз должен содержать:
- актуальное ТЗ;
- актуальный roadmap;
- запись в журнале релизов;
- актуальный transfer-файл для нового чата.
